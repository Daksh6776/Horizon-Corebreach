Critical Implementation Notes
1. Overview & Responsibilities
HorizonSkills fundamentally transforms the player character from a blank slate into a deeply simulated, highly specialized individual with physical limitations and mental health considerations.
Skill Progression: Expands the base skills in HorizonCore into 29 comprehensive disciplines. Actions in the world grant XP, leveling up skills to a cap of 100.
Specialization Paths: At levels 25 and 50, players must choose branching specializations (58 total paths) that define their character class, permanently locking out alternative branches.
Sports & Athletics: Provides 11 playable athletic minigames, allowing players and NPCs to compete for prestige, prizes, and physical stat buffs.
Psychology & Mental Health: Tracks the player's psychological state across 12 distinct conditions. Trauma from combat, addiction to consumables, and stress from overwork enforce realistic human limitations and require proactive treatment.

package com.horizonpack.horizonskills.core;

import com.horizonpack.horizonskills.data.attachments.SkillsAttachments;

import com.horizonpack.horizonskills.network.SkillsPacketHandler;

import com.horizonpack.horizonskills.systems.SportsEngine;
import net.neoforged.bus.api.IEventBus;
import net.neoforged.fml.ModContainer;
import net.neoforged.fml.common.Mod;
import net.neoforged.fml.config.ModConfig;
import net.neoforged.fml.event.lifecycle.FMLCommonSetupEvent;
import net.neoforged.neoforge.common.NeoForge;

@Mod(HorizonSkills.MODID)
public class HorizonSkills {
    public static final String MODID = "horizonskills";
    public static final String VERSION = "1.0.0";

    public HorizonSkills(IEventBus modEventBus, ModContainer modContainer) {
        modContainer.registerConfig(ModConfig.Type.COMMON, SkillsConfig.COMMON_SPEC);
        modContainer.registerConfig(ModConfig.Type.CLIENT, SkillsConfig.CLIENT_SPEC);

        SkillsRegistries.BLOCKS.register(modEventBus);
        SkillsRegistries.ITEMS.register(modEventBus);
        SkillsRegistries.BLOCK_ENTITIES.register(modEventBus);
        SkillsRegistries.MENU_TYPES.register(modEventBus);

        SkillsRegistries.SKILLS.register(modEventBus);
        SkillsRegistries.SPECIALIZATIONS.register(modEventBus);
        SkillsRegistries.SPORTS.register(modEventBus);
        SkillsRegistries.PSYCHOLOGY_STATES.register(modEventBus);

        SkillsAttachments.ATTACHMENT_TYPES.register(modEventBus);

        modEventBus.addListener(this::commonSetup);
        modEventBus.addListener(SkillsRegistries::registerNewRegistries);
        modEventBus.addListener(SkillsPacketHandler::register);

        NeoForge.EVENT_BUS.register(SkillsEventHandlers.class);
    }

    private void commonSetup(FMLCommonSetupEvent event) {
        event.enqueueWork(() -> {
            SkillEngine.init();
            PsychologyEngine.init();
            SportsEngine.init();
        });
    }
}

        event.register(new RegistryBuilder<>(SKILL_KEY).sync(true));
        event.register(new RegistryBuilder<>(SPECIALIZATION_KEY).sync(true));
        event.register(new RegistryBuilder<>(SPORT_KEY).sync(true));
        event.register(new RegistryBuilder<>(PSYCHOLOGY_KEY).sync(true));
    }

    static {
        HorizonSkill.registerAll();
        SkillSpecialization.registerAll();
        SportMinigame.registerAll();
        PsychologyState.registerAll();
    }
}

package com.horizonpack.horizonskills.systems;

import com.horizonpack.horizonskills.data.attachments.PlayerMindset;
import com.horizonpack.horizonskills.data.attachments.SkillsAttachments;
import net.minecraft.resources.ResourceLocation;
import net.minecraft.server.level.ServerPlayer;

public class PsychologyEngine {

    public static void init() {}

    public static void tickPsychology(ServerPlayer player) {
        PlayerMindset mind = player.getData(SkillsAttachments.MINDSET);
        
        // Passive stress decay if sleeping or content
        if (player.isSleeping() && mind.getStressLevel() > 0) {
            mind.setStressLevel(mind.getStressLevel() - 5);
        }

        // Apply debuffs based on state
        String state = mind.getCurrentState().getPath();
        if (state.equals("stressed")) {
            player.causeFoodExhaustion(0.01f); // Drains hunger
        } else if (state.equals("depressed")) {
            player.addEffect(new net.minecraft.world.effect.MobEffectInstance(net.minecraft.world.effect.MobEffects.MOVEMENT_SLOWDOWN, 100, 0, false, false));
        } else if (state.equals("manic")) {
            player.addEffect(new net.minecraft.world.effect.MobEffectInstance(net.minecraft.world.effect.MobEffects.DIG_SPEED, 100, 0, false, false));
        }
    }

    public static void applyTrauma(ServerPlayer player, int stressAmount) {
        PlayerMindset mind = player.getData(SkillsAttachments.MINDSET);
        mind.setStressLevel(mind.getStressLevel() + stressAmount);

        if (mind.getStressLevel() >= 100) {
            mind.setCurrentState(ResourceLocation.fromNamespaceAndPath("horizonskills", "traumatized"));
            player.displayClientMessage(net.minecraft.network.chat.Component.translatable("message.horizonskills.traumatized"), true);
        }
    }
}

package com.horizonpack.horizonskills.systems;

import net.minecraft.core.BlockPos;
import net.minecraft.server.level.ServerLevel;
import net.minecraft.world.entity.player.Player;

import java.util.HashMap;
import java.util.Map;
import java.util.UUID;

public class SportsEngine {

    // Track active games: GameUUID -> Minigame Instance
    private static final Map<UUID, ActiveMinigame> activeGames = new HashMap<>();

    public static void init() {}

    public static void startArcheryContest(ServerLevel level, BlockPos targetPos, Player initiator) {
        ActiveMinigame game = new ActiveMinigame("archery_contest", level, targetPos);
        game.addPlayer(initiator);
        activeGames.put(UUID.randomUUID(), game);
    }

    public static void recordArcheryHit(ServerLevel level, BlockPos targetPos, Player shooter, int points) {
        for (ActiveMinigame game : activeGames.values()) {
            if (game.getType().equals("archery_contest") && game.getOrigin().equals(targetPos)) {
                game.addScore(shooter, points);
                if (game.isComplete()) {
                    Player winner = game.getWinner();
                    SkillEngine.awardXp((net.minecraft.server.level.ServerPlayer) winner, "athletics", 500);
                    activeGames.remove(game.getId());
                }
                break;
            }
        }
    }

    public static class ActiveMinigame {
        private final UUID id = UUID.randomUUID();
        private final String type;
        private final ServerLevel level;
        private final BlockPos origin;
        private final Map<UUID, Integer> scores = new HashMap<>();
        private int totalShots = 0;

        public ActiveMinigame(String type, ServerLevel level, BlockPos origin) {
            this.type = type;
            this.level = level;
            this.origin = origin;
        }

        public void addPlayer(Player player) { scores.put(player.getUUID(), 0); }
        public void addScore(Player player, int points) { 
            scores.merge(player.getUUID(), points, Integer::sum); 
            totalShots++;
        }
        public boolean isComplete() { return totalShots >= 10; } // Game ends after 10 shots
        public String getType() { return type; }
        public BlockPos getOrigin() { return origin; }
        public UUID getId() { return id; }
        
        public Player getWinner() {
            UUID top = scores.entrySet().stream().max(Map.Entry.comparingByValue()).get().getKey();
            return level.getPlayerByUUID(top);
        }
    }
}

package com.horizonpack.horizonskills.events;

import com.horizonpack.horizonskills.core.HorizonSkills;
import com.horizonpack.horizonskills.systems.PsychologyEngine;
import com.horizonpack.horizonskills.systems.SkillEngine;
import net.minecraft.server.level.ServerPlayer;
import net.neoforged.bus.api.SubscribeEvent;
import net.neoforged.fml.common.Mod;
import net.neoforged.neoforge.event.entity.living.LivingDamageEvent;
import net.neoforged.neoforge.event.entity.player.PlayerEvent;
import net.neoforged.neoforge.event.level.BlockEvent;
import net.neoforged.neoforge.event.tick.PlayerTickEvent;

@Mod.EventBusSubscriber(modid = HorizonSkills.MODID)
public class SkillsEventHandlers {

    @SubscribeEvent
    public static void onBlockBreak(BlockEvent.BreakEvent event) {
        if (!event.getLevel().isClientSide() && event.getPlayer() instanceof ServerPlayer player) {
            // Mining XP
            if (event.getState().is(net.minecraft.tags.BlockTags.MINEABLE_WITH_PICKAXE)) {
                SkillEngine.awardXp(player, "mining", 10);
            }
            // Woodcutting XP
            if (event.getState().is(net.minecraft.tags.BlockTags.MINEABLE_WITH_AXE)) {
                SkillEngine.awardXp(player, "woodcutting", 10);
            }
        }
    }

    @SubscribeEvent
    public static void onPlayerDamaged(LivingDamageEvent.Post event) {
        if (event.getEntity() instanceof ServerPlayer player) {
            // If taking massive damage (>10), apply psychological trauma
            if (event.getNewDamage() > 10.0f) {
                PsychologyEngine.applyTrauma(player, 20); // Adds 20 stress
            }
        }
    }

    @SubscribeEvent
    public static void onPlayerTick(PlayerTickEvent.Post event) {
        if (event.getEntity() instanceof ServerPlayer player && player.tickCount % 100 == 0) {
            PsychologyEngine.tickPsychology(player);
        }
    }
}

27. Critical Implementation Notes
NeoForge 1.21 Data Attachments: The psychological state and locked specializations use the new NeoForgeRegistries.ATTACHMENT_TYPES rather than the old capability system. PlayerMindset implements INBTSerializable and uses .copyOnDeath() in the builder to ensure that trauma and addictions are not magically cured by respawning.
Dynamic Skill Evaluation Overhead: Modifying vanilla block break speed or damage based on 29 dynamic skills every tick is dangerous. SkillEngine caches the effective modifier of mining speed and combat_melee damage inside a transient map that only recalculates when a SkillLevelUpEvent is fired or a new SkillSpecialization is locked in.
Minigame Instances Memory Leaks: The SportsEngine.ActiveMinigame instances map uses UUIDs. If an NPC or player disconnects mid-game, the match will hang in memory indefinitely. A server-tick sweep must iterate activeGames every 1200 ticks (1 minute) to purge any match older than 10 real-time minutes that hasn't fired an update.
Specialization Branching Logic: When a player hits level 25 or 50, they are not forced to pick a spec immediately. The ChooseSpecializationPacket validates server-side that the required level has been achieved and that the skillId has no existing mutually-exclusive spec assigned in the PlayerSpecializations attachment.
Psychology Stress Degeneration: The PsychologyEngine alters FoodExhaustion based on the "stressed" state. This ties directly into horizonsurvival's DigestionEngine. Care must be taken to apply exhaustion cleanly via player.causeFoodExhaustion() rather than artificially altering the calorie float, preserving mod interaction integrity.
Archery Target Hit Computation: Vanilla TargetBlock uses basic AABB collisions. The ArcheryTargetBlock uses a ProjectileImpactEvent hook to measure the exact HitResult.getLocation() relative to the block's center coordinate, mathematically assigning a score from 1-10 to feed into the SportsEngine.recordArcheryHit() method.
Therapy Item Cooldown: The TherapyJournalItem reduces stress but enforces a heavy 24,000 tick (1 in-game day) item cooldown via Player#getCooldowns().addCooldown(). Bypassing this cooldown would trivialize the entire HorizonPsychology simulation layer.
Javelin Entity Physics: The JavelinItem throws a custom JavelinEntity extending AbstractArrow. Unlike arrows, its drag coefficient and gravity drop curve are dynamically altered upon launch based on the thrower's athletics skill level queried through HorizonCore.
