1. Overview & Responsibilities
HorizonEconomy simulates the intricate web of macro and microeconomics spanning human history. It introduces systems for resource valuation, currency evolution, labor management, and advanced agricultural scaling.
Currency Evolution: Tracks exchange mediums from primitive barter and commodity money (salt, grain) to minted coinage, fiat paper, and digital currencies.
Global Commodity Markets: Tracks the supply, demand, and precise historical value of 13 commodity categories across the world.
Financial Institutions: Functional blocks that evolve the financial capabilities of a settlement (Markets → Banks → Stock Exchanges).
Labor & Wages: A dynamic employment system managing 6 labor tiers, worker happiness, strikes, and wage distribution.
Agricultural Systems: Progressive farming techniques (slash-and-burn to hydroponics) that govern crop yields, soil depletion, and field rotation on a macro scale.

import net.neoforged.fml.event.lifecycle.FMLCommonSetupEvent;
import net.neoforged.neoforge.common.NeoForge;

@Mod(HorizonEconomy.MODID)
public class HorizonEconomy {
    public static final String MODID = "horizoneconomy";
    public static final String VERSION = "1.0.0";

    public HorizonEconomy(IEventBus modEventBus, ModContainer modContainer) {
        modContainer.registerConfig(ModConfig.Type.COMMON, EconomyConfig.COMMON_SPEC);
        modContainer.registerConfig(ModConfig.Type.CLIENT, EconomyConfig.CLIENT_SPEC);

        EconomyRegistries.BLOCKS.register(modEventBus);
        EconomyRegistries.ITEMS.register(modEventBus);
        EconomyRegistries.BLOCK_ENTITIES.register(modEventBus);
        EconomyRegistries.ENTITY_TYPES.register(modEventBus);
        EconomyRegistries.MENU_TYPES.register(modEventBus);

        EconomyRegistries.CURRENCY_TYPES.register(modEventBus);
        EconomyRegistries.COMMODITY_CATEGORIES.register(modEventBus);
        EconomyRegistries.LABOR_TYPES.register(modEventBus);
        EconomyRegistries.AGRICULTURAL_SYSTEMS.register(modEventBus);

        modEventBus.addListener(this::commonSetup);
        modEventBus.addListener(EconomyRegistries::registerNewRegistries);
        modEventBus.addListener(EconomyRegistries::registerEntityAttributes);
        modEventBus.addListener(EconomyPacketHandler::register);

        NeoForge.EVENT_BUS.register(EconomyEventHandlers.class);
    }

    private void commonSetup(FMLCommonSetupEvent event) {
        event.enqueueWork(() -> {
            MarketEngine.init();
        });
    }
}

10. Global Market Engine
File: systems/MarketEngine.java
Handles the background ticking of price volatility based on global supply/demand shocks from natural disasters (HorizonWorld) or wars (HorizonSociety).
code
Java
package com.horizonpack.horizoneconomy.systems;

import com.horizonpack.horizoneconomy.data.EconomySavedData;
import com.horizonpack.horizoneconomy.data.registry.CommodityCategory;
import net.minecraft.resources.ResourceLocation;
import net.minecraft.server.level.ServerLevel;

import java.util.Map;
import java.util.Random;

public class MarketEngine {

    public static void init() {}

    public static void tickMarket(ServerLevel level) {
        EconomySavedData data = EconomySavedData.get(level);
        Random rand = level.getRandom();
        
        // Apply random volatility walk to all commodities
        for (Map.Entry<ResourceLocation, Double> entry : data.getGlobalPrices().entrySet()) {
            CommodityCategory category = getCategoryOrNull(level, entry.getKey());
            if (category == null) continue;
            
            double currentPrice = entry.getValue();
            double volatility = category.priceVolatility();
            
            // Random walk: -volatility% to +volatility%
            double shift = 1.0 + ((rand.nextDouble() * 2.0 - 1.0) * volatility);
            double newPrice = Math.max(category.basePrice() * 0.1, currentPrice * shift); // Price floor
            
            // Revert gently to base price over time
            newPrice += (category.basePrice() - newPrice) * 0.05; 
            
            data.setGlobalPrice(entry.getKey(), newPrice);
        }
    }
    
    private static CommodityCategory getCategoryOrNull(ServerLevel level, ResourceLocation id) {
        return level.registryAccess().registryOrThrow(com.horizonpack.horizoneconomy.core.EconomyRegistries.COMMODITY_KEY).get(id);
    }
}
11. World SavedData (Economy State)
File: data/EconomySavedData.java
Stores global prices, civilization bank reserves, and player bank accounts. Ensures long values are used to prevent precision loss with fiat inflation.
code
Java
package com.horizonpack.horizoneconomy.data;

import net.minecraft.nbt.CompoundTag;
import net.minecraft.nbt.ListTag;
import net.minecraft.nbt.Tag;
import net.minecraft.resources.ResourceLocation;
import net.minecraft.server.level.ServerLevel;
import net.minecraft.world.level.saveddata.SavedData;
import org.jetbrains.annotations.NotNull;

import java.util.HashMap;
import java.util.Map;
import java.util.UUID;

public class EconomySavedData extends SavedData {

    public static final String DATA_NAME = "horizoneconomy_state";

    private final Map<ResourceLocation, Double> globalPrices = new HashMap<>();
    private final Map<UUID, Long> bankAccounts = new HashMap<>(); // UUID can be Player or Civilization
    private double globalInflationRate = 1.0;

    public static EconomySavedData get(ServerLevel level) {
        return level.getDataStorage().computeIfAbsent(
                new SavedData.Factory<>(EconomySavedData::new, EconomySavedData::load),
                DATA_NAME
        );
    }

    public Map<ResourceLocation, Double> getGlobalPrices() { return globalPrices; }
    
    public void setGlobalPrice(ResourceLocation id, double price) {
        globalPrices.put(id, price);
        setDirty();
    }

    public long getBalance(UUID id) { return bankAccounts.getOrDefault(id, 0L); }

    public void adjustBalance(UUID id, long amount) {
        bankAccounts.put(id, getBalance(id) + amount);
        setDirty();
    }
    
    public double getInflationRate() { return globalInflationRate; }
    public void setInflationRate(double rate) { this.globalInflationRate = rate; setDirty(); }

    @Override
    public @NotNull CompoundTag save(@NotNull CompoundTag tag) {
        CompoundTag pricesTag = new CompoundTag();
        globalPrices.forEach((id, price) -> pricesTag.putDouble(id.toString(), price));
        tag.put("Prices", pricesTag);

        ListTag accountsTag = new ListTag();
        bankAccounts.forEach((uuid, balance) -> {
            CompoundTag acc = new CompoundTag();
            acc.putUUID("UUID", uuid);
            acc.putLong("Balance", balance);
            accountsTag.add(acc);
        });
        tag.put("Accounts", accountsTag);
        tag.putDouble("Inflation", globalInflationRate);
        return tag;
    }

    public static EconomySavedData load(CompoundTag tag) {
        EconomySavedData data = new EconomySavedData();
        CompoundTag pricesTag = tag.getCompound("Prices");
        for (String key : pricesTag.getAllKeys()) {
            data.globalPrices.put(ResourceLocation.parse(key), pricesTag.getDouble(key));
        }

        ListTag accountsTag = tag.getList("Accounts", Tag.TAG_COMPOUND);
        for (int i = 0; i < accountsTag.size(); i++) {
            CompoundTag acc = accountsTag.getCompound(i);
            data.bankAccounts.put(acc.getUUID("UUID"), acc.getLong("Balance"));
        }
        
        data.globalInflationRate = tag.contains("Inflation") ? tag.getDouble("Inflation") : 1.0;
        return data;
    }
}

code

package com.horizonpack.horizoneconomy.network.packets;

import com.horizonpack.horizoneconomy.core.HorizonEconomy;
import com.horizonpack.horizoneconomy.data.EconomySavedData;
import io.netty.buffer.ByteBuf;
import net.minecraft.network.codec.ByteBufCodecs;
import net.minecraft.network.codec.StreamCodec;
import net.minecraft.network.protocol.common.custom.CustomPacketPayload;
import net.minecraft.resources.ResourceLocation;
import net.neoforged.neoforge.network.handling.IPayloadContext;

public record MarketTransactionPacket(ResourceLocation commodityId, int amount, boolean isBuy) implements CustomPacketPayload {

    public static final Type<MarketTransactionPacket> TYPE = new Type<>(ResourceLocation.fromNamespaceAndPath(HorizonEconomy.MODID, "market_transaction"));

    public static final StreamCodec<ByteBuf, MarketTransactionPacket> STREAM_CODEC = StreamCodec.composite(
        ResourceLocation.STREAM_CODEC, MarketTransactionPacket::commodityId,
        ByteBufCodecs.INT, MarketTransactionPacket::amount,
        ByteBufCodecs.BOOL, MarketTransactionPacket::isBuy,
        MarketTransactionPacket::new
    );

    @Override
    public Type<MarketTransactionPacket> type() { return TYPE; }

    public static void handleServer(MarketTransactionPacket packet, IPayloadContext context) {
        context.enqueueWork(() -> {
            net.minecraft.server.level.ServerPlayer player = (net.minecraft.server.level.ServerPlayer) context.player();
            EconomySavedData data = EconomySavedData.get(player.serverLevel());
            double price = data.getGlobalPrices().getOrDefault(packet.commodityId(), 10.0);
            long totalCost = (long)(price * packet.amount());
            
            long balance = data.getBalance(player.getUUID());
            if (packet.isBuy() && balance >= totalCost) {
                data.adjustBalance(player.getUUID(), -totalCost);
                // Give commodity item to player inventory...
            } else if (!packet.isBuy()) {
                // Consume commodity item, give cash
                data.adjustBalance(player.getUUID(), totalCost);
            }
        });
    }
}

26. Critical Implementation Notes
Floating Point Precision for Fiat Currency:
Never use float or double for direct bank account balances, as fiat currencies inherently inflate to thousands or millions of units causing precision loss. The BankBlockEntity and EconomySavedData must strictly utilize long values to represent atomic currency units, with UI overlays mapping it against the globalInflationRate.
Market Evaluation Threading:
Calculating supply and demand against millions of simulated units inside MarketEngine cannot be handled on the main server thread. MarketEngine.tickMarket() should farm intensive calculations to net.minecraft.Util.backgroundExecutor() using CompletableFuture, then sync results back via thread-safe atomic updates to the EconomySavedData.
Data Component Migration for NBT Items:
Items like CommodityContractItem or CreditCardItem inherently store unique balances and ownership hashes. Under NeoForge 1.21.1, do not use getOrCreateTag(). Define a Custom Data Component via DeferredRegister.DataComponents to strictly serialize these values safely and ensure stack comparisons behave.
Offline Economy Simulation:
To ensure the global economy does not stagnate while chunks are unloaded, EconomySavedData relies entirely on a day-tick mathematical update rather than specific chunk entities. Stock crashes, inflation, and demand shocks apply mathematically on the DayAdvanceEvent rather than waiting for players to load specific cities.
Historical Forced Labor Ethics (ESRB Compliance):
The forced_labor_system commodity exists mechanically as it was the reality of the ancient/medieval eras. However, the isEthicallyRestricted flag exists in CommodityCategory. Under horizonsociety, any Civilization with the Democracy or Republic type will permanently default to -100 Hostile Reputation if a neighboring empire or player engages in this commodity, triggering emancipatory declarations of war. Furthermore, the allowForcedLabor boolean inside EconomyConfig allows server operators to completely disable this layer, converting those economic nodes into generic unskilled_labor.
Labor Strike Cascade:
Labor forces check their assigned wages vs global inflation. If real wages dip beneath the baseDailyWage threshold dictated by the LaborType registry, the StrikeStartEvent triggers, instantly halting MachineBlockEntity and AgriculturalSystem yields in the affected region.
Agricultural Controller & Chunk Loading:
The IrrigationControllerBlockEntity scans connected horizonbiome:crop instances up to a radius of 64 blocks to apply the AgriculturalSystem modifier. Given chunk loading limits, this scan must abort or pause gracefully if the adjacent farm chunk unloads to prevent OutOfMemory chunk-thrashing leaks.
Merchant Caravan Pathfinding:
The MerchantCaravanEntity must safely pathfind distances larger than loaded chunks. This is handled by transferring the entity to a virtual TradeNetworkEngine (despawning it) while it crosses unloaded chunks, then respawning the entity at the destination settlement border upon arrival calculation.
