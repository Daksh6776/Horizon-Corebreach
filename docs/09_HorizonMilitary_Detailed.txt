1. Overview & Responsibilities
HorizonMilitary introduces a highly advanced warfare, justice, and emergency response simulation to the Horizon Corebreach modpack.
It replaces vanilla combat interactions with disciplined armies, strategic sieges, supply lines, and organized crime networks.
Warfare & Armies: 18 distinct military unit types ranging from ancient militia to modern special forces. Units operate in strict formations utilizing 19 specific weapon types.
Sieges & Fortifications: 10 tiers of defensive structures and 8 distinct siege engines. Destructibility is governed by the structural integrity engine from HorizonData.
Logistics: Armies suffer attrition without supply lines. Ammunition, food, and medical supplies must be transported to the front lines.
Crime & Justice: 10 defined crime types with detection, punishment, and recidivism mechanics. Settlements spawn thieves, smugglers, and syndicates based on poverty and stability.
Emergency Response: Firefighters, paramedics, and disaster relief systems that react to events fired by HorizonWorld's DisasterEngine.

import net.neoforged.bus.api.IEventBus;
import net.neoforged.fml.ModContainer;
import net.neoforged.fml.common.Mod;
import net.neoforged.fml.config.ModConfig;
import net.neoforged.fml.event.lifecycle.FMLCommonSetupEvent;
import net.neoforged.neoforge.common.NeoForge;

@Mod(HorizonMilitary.MODID)
public class HorizonMilitary {
    public static final String MODID = "horizonmilitary";
    public static final String VERSION = "1.0.0";

    public HorizonMilitary(IEventBus modEventBus, ModContainer modContainer) {
        modContainer.registerConfig(ModConfig.Type.COMMON, MilitaryConfig.COMMON_SPEC);
        modContainer.registerConfig(ModConfig.Type.CLIENT, MilitaryConfig.CLIENT_SPEC);

        MilitaryRegistries.BLOCKS.register(modEventBus);
        MilitaryRegistries.ITEMS.register(modEventBus);
        MilitaryRegistries.BLOCK_ENTITIES.register(modEventBus);
        MilitaryRegistries.ENTITY_TYPES.register(modEventBus);
        MilitaryRegistries.MENU_TYPES.register(modEventBus);
        MilitaryRegistries.CRIME_TYPES.register(modEventBus);
        MilitaryRegistries.DATA_COMPONENT_TYPES.register(modEventBus);

        modEventBus.addListener(this::commonSetup);
        modEventBus.addListener(MilitaryRegistries::registerNewRegistries);
        modEventBus.addListener(UnitRegistry::registerEntityAttributes);
        modEventBus.addListener(SiegeRegistry::registerEntityAttributes);
        modEventBus.addListener(MilitaryPacketHandler::register);

        NeoForge.EVENT_BUS.register(MilitaryEventHandlers.class);
    }

    private void commonSetup(FMLCommonSetupEvent event) {
        event.enqueueWork(() -> {
            CrimeEngine.init();
            FormationManager.init();
            LogisticsEngine.init();
            EmergencyEngine.init();
        });
    }
}

package com.horizonpack.horizonmilitary.systems;

import com.horizonpack.horizonmilitary.entities.units.MilitaryUnitEntity;
import net.minecraft.core.BlockPos;
import net.minecraft.server.level.ServerLevel;
import net.minecraft.world.phys.Vec3;

import java.util.ArrayList;
import java.util.List;
import java.util.UUID;

public class FormationManager {

    // Simplified list of active formations
    private static final List<Formation> activeFormations = new ArrayList<>();

    public static void init() {}

    public static Formation createFormation(UUID commanderId, FormationType type) {
        Formation f = new Formation(commanderId, type);
        activeFormations.add(f);
        return f;
    }

    public static void tickFormations(ServerLevel level) {
        for (Formation f : activeFormations) {
            f.tick(level);
        }
    }

    public enum FormationType {
        LINE, COLUMN, WEDGE, SQUARE, SKIRMISH
    }

    public static class Formation {
        private final UUID commanderId;
        private final FormationType type;
        private final List<MilitaryUnitEntity> units = new ArrayList<>();
        private BlockPos targetDestination;
        private float facing;

        public Formation(UUID commanderId, FormationType type) {
            this.commanderId = commanderId;
            this.type = type;
        }

        public void addUnit(MilitaryUnitEntity unit) {
            units.add(unit);
            unit.setFormation(this);
        }

        public void removeUnit(MilitaryUnitEntity unit) {
            units.remove(unit);
            unit.setFormation(null);
        }

        public void orderMove(BlockPos destination, float facing) {
            this.targetDestination = destination;
            this.facing = facing;
            assignPositions();
        }

        private void assignPositions() {
            if (targetDestination == null || units.isEmpty()) return;

            int count = units.size();
            int spacing = 2; // blocks between units

            switch (type) {
                case LINE -> {
                    // Assign positions in a line perpendicular to facing
                    int width = count;
                    int startX = -(width * spacing) / 2;
                    for (int i = 0; i < count; i++) {
                        Vec3 offset = new Vec3(startX + (i * spacing), 0, 0).yRot((float) Math.toRadians(-facing));
                        BlockPos unitTarget = targetDestination.offset((int) offset.x, 0, (int) offset.z);
                        units.get(i).setFormationTarget(unitTarget);
                    }
                }
                case COLUMN -> {
                    // Assign positions in a column parallel to facing
                     for (int i = 0; i < count; i++) {
                        Vec3 offset = new Vec3(0, 0, -(i * spacing)).yRot((float) Math.toRadians(-facing));
                        BlockPos unitTarget = targetDestination.offset((int) offset.x, 0, (int) offset.z);
                        units.get(i).setFormationTarget(unitTarget);
                    }
                }
                // Implement WEDGE, SQUARE, SKIRMISH similarly
                default -> {
                    for (MilitaryUnitEntity unit : units) {
                        unit.setFormationTarget(targetDestination); // Fallback
                    }
                }
            }
        }

        public void tick(ServerLevel level) {
            // Verify cohesion, adjust speed for slowest unit, handle engaged combat
            if (!units.isEmpty() && targetDestination != null) {
                boolean allInPosition = true;
                for (MilitaryUnitEntity unit : units) {
                    if (unit.blockPosition().distSqr(unit.getFormationTarget()) > 4) {
                        allInPosition = false;
                        break;
                    }
                }
                // If all in position, formation holds. If engaged, execute formation attacks (e.g., volley fire)
            }
        }
    }

25. Critical Implementation Notes
Formation Tick Optimization: FormationManager.tickFormations() iterates over all active formations every tick. To prevent server lag, position assignment (assignPositions()) must only be called when the formation's target destination or facing changes, not every tick. The per-unit tick should only check cohesion distance.
Siege Engine Destruction: FortificationBlock integrates directly with HorizonData's StructuralIntegrity system. When a Trebuchet or Cannon projectile hits a wall, it does not use Level#destroyBlock. It applies a massive load spike to the targeted block, forcing the structural integrity engine to calculate an authentic collapse pattern for the surrounding wall section.
Logistics Attrition Radius: LogisticsEngine checks if units are within supply range. Scanning the level for roads/rails (HorizonPower infrastructure) every hour for every unit is too expensive. Instead, settlements project a "supply aura" radius stored in a spatial hash map. Units check their intersection with this aura.
Firearm Reload Mechanics: FirearmItem uses NeoForge data components to track loaded ammunition and reload progress. The reload action is a channeled use action (like drawing a bow). If interrupted, the progress is lost. This enforces realistic musket/rifle firing lines.
Crime Pathfinding Constraints: When CrimeEngine spawns a thief or assassin, they must use a customized navigation goal that actively avoids the line of sight of PoliceStationBlockEntity guards and DefensiveTowerBlock watchmen.
Firefighter Water Supply: FirefighterEntity using ExtinguishFireGoal requires nearby water sources (or a fire hydrant/pump from HorizonUrbanization if implemented). If no water is within 32 blocks, they will fail to extinguish the fire and run back to the station.
Cross-Mod Tiers: WeaponRegistry melee weapons utilize vanilla Tiers for base damage calculations, but their actual damage output is heavily modified by HorizonData material properties (e.g., a Copper Sword breaks faster and does less damage than a Bronze Sword, even if both map to Tiers.STONE).
Entity Alignment/Loyalty: MilitaryUnitEntity contains a civilizationId UUID in its NBT. Units will actively attack units with a different civilizationId if their respective Civilizations (from HorizonSociety) are currently at war. This lookup caches the war status to avoid constantly querying SocietySavedData.
