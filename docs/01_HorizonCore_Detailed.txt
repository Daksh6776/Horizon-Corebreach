## 1. Overview & Responsibilities

HorizonCore is the **single required dependency** of every other Horizon mod. It provides:

- A **central event bus bridge** so sub-mods communicate without direct class references
- **Custom registries** for technologies, skills, professions, nutrients, diseases, weather types, and disasters
- The **Age/Era progression system** (Stone → Copper → Bronze → Iron → Medieval → Industrial → Modern)
- The **Technology Tree** — a directed acyclic graph of researchable technologies gated behind age, resources, and prerequisite techs
- The **Skill system** — per-player XP-based leveling across dozens of practical disciplines
- The **Science & Research system** — laboratory research that consumes time and materials to unlock advances
- The **Education system** — institutions (apprenticeships, schools, universities, labs) that accelerate research and skill gain
- The **player capability** (`IHorizonPlayerData`) — the single NBT-serializable object attached to every player storing nutrition, skills, unlocked techs, body temperature, language proficiencies, and more
- **World-level SavedData** (`WorldAgeData`) tracking global day count, per-civilization days, and per-settlement age overrides
- All **cross-mod events** that sub-mods fire and listen to
- **Network packets** for syncing capability data to the client
- The **HUD overlay** showing age, active buffs/debuffs, and key stats

## 3. Mod Entry Point

**File:** `core/HorizonCore.java`

```java
package com.horizonpack.horizoncore.core;

import net.neoforged.bus.api.IEventBus;
import net.neoforged.fml.ModContainer;
import net.neoforged.fml.common.Mod;
import net.neoforged.fml.config.ModConfig;
import net.neoforged.fml.event.lifecycle.FMLCommonSetupEvent;
import net.neoforged.neoforge.common.NeoForge;

@Mod(HorizonCore.MODID)
public class HorizonCore {
    public static final String MODID = "horizoncore";
    public static final String VERSION = "1.0.0";

    public HorizonCore(IEventBus modEventBus, ModContainer modContainer) {
        // Register configs
        modContainer.registerConfig(ModConfig.Type.COMMON, HorizonConfig.COMMON_SPEC);
        modContainer.registerConfig(ModConfig.Type.CLIENT, HorizonConfig.CLIENT_SPEC);

        // Register all deferred registers onto the mod event bus
        HorizonRegistries.TECHNOLOGIES.register(modEventBus);
        HorizonRegistries.SKILLS.register(modEventBus);
        HorizonRegistries.PROFESSIONS.register(modEventBus);
        HorizonRegistries.NUTRIENTS.register(modEventBus);
        HorizonRegistries.DISEASES.register(modEventBus);
        HorizonRegistries.WEATHER_TYPES.register(modEventBus);
        HorizonRegistries.DISASTERS.register(modEventBus);
        HorizonRegistries.BLOCKS.register(modEventBus);
        HorizonRegistries.BLOCK_ENTITIES.register(modEventBus);
        HorizonRegistries.ITEMS.register(modEventBus);
        HorizonRegistries.MENU_TYPES.register(modEventBus);

        // Lifecycle listeners on the mod bus
        modEventBus.addListener(this::commonSetup);
        modEventBus.addListener(HorizonCapabilities::register);
        modEventBus.addListener(HorizonPacketHandler::register);

        // Game event listeners on the NeoForge bus
        NeoForge.EVENT_BUS.register(HorizonEventHandlers.class);
    }

    private void commonSetup(FMLCommonSetupEvent event) {
        event.enqueueWork(() -> {
            HorizonRegistries.init();
            TechnologyGraph.buildGraph();     // Build the DAG after all techs are registered
        });
    }
}
```

## 5. Age & Era System

### 5.1 HorizonAge Enum

**File:** `data/HorizonAge.java`

The seven ages of civilization. Each age has a day range, a string ID for serialization, a human-readable display name, and a list of gameplay restrictions it lifts.

```java
package com.horizonpack.horizoncore.data;

import net.minecraft.util.StringRepresentable;

public enum HorizonAge implements StringRepresentable {
    // id,           startDay, endDay,  serialId,      displayName
    STONE_AGE       (0,        5000,    "stone_age",   "Stone Age"),
    COPPER_AGE      (5000,     10000,   "copper_age",  "Copper Age"),
    BRONZE_AGE      (10000,    15000,   "bronze_age",  "Bronze Age"),
    IRON_AGE        (15000,    30000,   "iron_age",    "Iron Age"),
    MEDIEVAL        (30000,    50000,   "medieval",    "Medieval Era"),
    INDUSTRIAL      (50000,    75000,   "industrial",  "Industrial Revolution"),
    MODERN          (75000,    Integer.MAX_VALUE, "modern", "Modern Era");

    private final int startDay;
    private final int endDay;
    private final String id;
    private final String displayName;

    HorizonAge(int startDay, int endDay, String id, String displayName) {
        this.startDay = startDay;
        this.endDay   = endDay;
        this.id       = id;
        this.displayName = displayName;
    }

    /** Returns the age that corresponds to a given global day count. */
    public static HorizonAge fromDay(int day) {
        for (HorizonAge age : values()) {
            if (day >= age.startDay && day < age.endDay) return age;
        }
        return MODERN;
    }

    /** True if this age is at least as advanced as the required age. */
    public boolean isAtLeast(HorizonAge required) {
        return this.ordinal() >= required.ordinal();
    }

    /** True if this age is strictly before the given age. */
    public boolean isBefore(HorizonAge other) {
        return this.ordinal() < other.ordinal();
    }

    public int getStartDay()      { return startDay; }
    public int getEndDay()        { return endDay; }
    public String getDisplayName(){ return displayName; }
    public int getDaySpan()       { return endDay == Integer.MAX_VALUE ? -1 : endDay - startDay; }

    @Override
    public String getSerializedName() { return id; }
}
```

### 5.2 Age Transition Logic

The global age is computed purely from the day counter held in `WorldAgeData`. When a new day ticks and the day crosses an age threshold, an `AgeAdvancementEvent` is fired on the NeoForge event bus. All other mods listen to that event and update their own state (unlocking new crafting stations, new mob spawns, new biome features, etc.).

Individual civilizations and settlements can be ahead of or behind the global age. `WorldAgeData` stores their own day counters independently.

**Age config multiplier:** `daysPerAgeMultiplier` in `common.toml` scales all `startDay`/`endDay` values at runtime, allowing server admins to speed up or slow down progression.

### 5.3 Age-Gated Recipe Condition

**File:** `utils/AgeRecipeCondition.java`

Recipes in any Horizon mod can include a condition that prevents them from appearing unless the world is at or above a required age.

```java
package com.horizonpack.horizoncore.utils;

import com.horizonpack.horizoncore.data.HorizonAge;
import com.horizonpack.horizoncore.data.WorldAgeData;
import com.mojang.serialization.Codec;
import com.mojang.serialization.codecs.RecordCodecBuilder;
import net.neoforged.neoforge.common.conditions.ICondition;

public record AgeRecipeCondition(HorizonAge requiredAge) implements ICondition {

    public static final Codec<AgeRecipeCondition> CODEC = RecordCodecBuilder.create(inst -> inst.group(
        HorizonAge.CODEC.fieldOf("required_age").forGetter(AgeRecipeCondition::requiredAge)
    ).apply(inst, AgeRecipeCondition::new));

    @Override
    public boolean test(IContext context) {
        // Evaluated on the server when the recipe manager is built
        // WorldAgeData is not available during datagen; return true in datagen context
        if (context == ICondition.IContext.EMPTY) return true;
        var level = context.getLevel();
        if (level == null) return true;
        return WorldAgeData.get(level.getServer().overworld())
                           .getGlobalAge()
                           .isAtLeast(requiredAge);
    }

    @Override
    public Codec<? extends ICondition> codec() { return CODEC; }
}
```

```java
package com.horizonpack.horizoncore.data;

import net.minecraft.resources.ResourceLocation;
import net.minecraft.world.item.ItemStack;

import java.util.List;

public class TechnologyData {
    private final ResourceLocation id;

    private final List<TechEffect> effects;         // What this tech unlocks
    private final String descriptionKey;            // Lang key for description
    private final String iconPath;                  // Texture path for the tech tree GUI

    public TechnologyData(Builder builder) {
        this.id                 = builder.id;
        this.requiredAge        = builder.requiredAge;

        this.descriptionKey     = builder.descriptionKey;
        this.iconPath           = builder.iconPath;
    }

    // Getters
    public ResourceLocation getId()                     { return id; }
    public HorizonAge getRequiredAge()                  { return requiredAge; }
    public List<ResourceLocation> getPrerequisites()    { return prerequisites; }
    public int getResearchTimeTicks()                   { return researchTimeTicks; }
    public List<ItemStack> getResearchCost()            { return researchCost; }
    public List<TechEffect> getEffects()                { return effects; }
    public String getDescriptionKey()                   { return descriptionKey; }
    public String getIconPath()                         { return iconPath; }

    /** Sealed interface for what a technology unlocks. */
    public sealed interface TechEffect permits
        TechEffect.UnlockRecipeCategory,
        TechEffect.UnlockBlock,
        TechEffect.UnlockSkillCap,
        TechEffect.ModifyAttribute,
        TechEffect.FireEvent {

        record UnlockRecipeCategory(ResourceLocation categoryId) implements TechEffect {}
        record UnlockBlock(ResourceLocation blockId) implements TechEffect {}
        record UnlockSkillCap(SkillType skill, int newMaxLevel) implements TechEffect {}
        record ModifyAttribute(ResourceLocation attributeId, double amount) implements TechEffect {}
        record FireEvent(String eventKey) implements TechEffect {}
    }

    public static Builder builder(ResourceLocation id) { return new Builder(id); }

        private List<ResourceLocation> prerequisites = List.of();
```

### 6.2 TechnologyGraph (DAG)

**File:** `data/TechnologyGraph.java`

After all mods have registered their technologies, `TechnologyGraph.buildGraph()` is called in `commonSetup`. It validates that all prerequisite references are valid and builds an adjacency list for efficient traversal.

```java
package com.horizonpack.horizoncore.data;

import com.horizonpack.horizoncore.core.HorizonRegistries;
import net.minecraft.resources.ResourceLocation;

import java.util.*;

public class TechnologyGraph {

    // adjacency: tech → list of techs that depend on it
    private static final Map<ResourceLocation, List<ResourceLocation>> dependents = new HashMap<>();
    // reverse: tech → its prerequisites
    private static final Map<ResourceLocation, List<ResourceLocation>> prerequisites = new HashMap<>();

    public static void buildGraph() {
        dependents.clear();
        prerequisites.clear();

        // Iterate every registered TechnologyData
        HorizonRegistries.TECHNOLOGIES.getEntries().forEach(entry -> {
            ResourceLocation id = entry.getKey().location();
            List<ResourceLocation> prereqs = entry.getValue().getPrerequisites();

            prerequisites.put(id, prereqs);

            for (ResourceLocation prereq : prereqs) {
                dependents.computeIfAbsent(prereq, k -> new ArrayList<>()).add(id);
            }
        });

        validateNoCycles();
    }

    /** Returns all technologies that are immediately unlockable given a set of already-unlocked techs. */
    public static List<ResourceLocation> getAvailableTechs(Set<ResourceLocation> unlocked, HorizonAge currentAge) {
        List<ResourceLocation> available = new ArrayList<>();
        HorizonRegistries.TECHNOLOGIES.getEntries().forEach(entry -> {
            ResourceLocation id = entry.getKey().location();
            TechnologyData tech = entry.getValue();
            if (unlocked.contains(id)) return;
            if (!currentAge.isAtLeast(tech.getRequiredAge())) return;
            if (unlocked.containsAll(tech.getPrerequisites())) available.add(id);
        });
        return available;
    }

    /** Returns all techs that this tech directly unlocks (i.e., techs that list it as a prerequisite). */
    public static List<ResourceLocation> getDependents(ResourceLocation techId) {
        return dependents.getOrDefault(techId, List.of());
    }

    private static void validateNoCycles() {
```

### 7.2 SkillDefinition (Registry Object)

**File:** `data/SkillDefinition.java`

```java
package com.horizonpack.horizoncore.data;

import net.minecraft.resources.ResourceLocation;
import java.util.Map;

public class SkillDefinition {
    private final SkillType type;
    private final int maxLevel;
    private final int baseXpPerLevel;    // XP needed for level 1. Level N costs base * N^1.5
    private final Map<Integer, List<ResourceLocation>> levelUpUnlocks; // level → list of tech IDs to auto-unlock

    public SkillDefinition(SkillType type, int maxLevel, int baseXpPerLevel,
                           Map<Integer, List<ResourceLocation>> levelUpUnlocks) {
        this.type             = type;
        this.maxLevel         = maxLevel;
        this.baseXpPerLevel   = baseXpPerLevel;
        this.levelUpUnlocks   = Map.copyOf(levelUpUnlocks);
    }

    /** XP required to go from level (n-1) to level n. */
    public int xpForLevel(int level) {
        return (int)(baseXpPerLevel * Math.pow(level, 1.5));
    }

    /** Total cumulative XP required to reach a given level from 0. */
    public int totalXpForLevel(int level) {
        int total = 0;
        for (int i = 1; i <= level; i++) total += xpForLevel(i);
        return total;
    }

    public SkillType getType()          { return type; }
    public int getMaxLevel()            { return maxLevel; }
    public int getBaseXpPerLevel()      { return baseXpPerLevel; }
    public Map<Integer, List<ResourceLocation>> getLevelUpUnlocks() { return levelUpUnlocks; }
}
```

## 8. Science & Research System

### 8.1 ResearchProject (Active Instance)

**File:** `data/ResearchProject.java`

When a player begins researching a technology at a Research Bench, a `ResearchProject` is created and stored in the `ResearchBenchBlockEntity`. It tracks progress in ticks and applies bonuses from nearby Libraries/Universities.

```java
package com.horizonpack.horizoncore.data;

import net.minecraft.nbt.CompoundTag;
import net.minecraft.resources.ResourceLocation;
import net.neoforged.neoforge.common.util.INBTSerializable;

public class ResearchProject implements INBTSerializable<CompoundTag> {

    private ResourceLocation technologyId;
    private int totalTicksRequired;
    private int ticksCompleted;
    private float speedMultiplier = 1.0f; // Modified by Library/University proximity

    public ResearchProject() {}

    public ResearchProject(ResourceLocation technologyId, int totalTicksRequired) {
        this.technologyId       = technologyId;
        this.totalTicksRequired = totalTicksRequired;
    }

    /** Called every tick by the ResearchBenchBlockEntity. Returns true when complete. */
    public boolean tick() {
        ticksCompleted += (int)(speedMultiplier);
        // Fractional remainder handled separately
        return ticksCompleted >= totalTicksRequired;
    }

    public float getProgress()              { return (float) ticksCompleted / totalTicksRequired; }
    public ResourceLocation getTechnologyId(){ return technologyId; }
    public void setSpeedMultiplier(float m) { this.speedMultiplier = m; }
    public float getSpeedMultiplier()       { return speedMultiplier; }

    @Override
    public CompoundTag serializeNBT() {
        CompoundTag tag = new CompoundTag();
        tag.putString("TechId", technologyId.toString());
        tag.putInt("TotalTicks", totalTicksRequired);
        tag.putInt("TicksDone", ticksCompleted);
        tag.putFloat("SpeedMult", speedMultiplier);
        return tag;
    }

    @Override
    public void deserializeNBT(CompoundTag tag) {
        technologyId       = new ResourceLocation(tag.getString("TechId"));
        totalTicksRequired = tag.getInt("TotalTicks");
        ticksCompleted     = tag.getInt("TicksDone");
        speedMultiplier    = tag.getFloat("SpeedMult");
    }
}
```

## 9. Education System

### 9.1 Institution Blocks

**Research Bench** — The core crafting station for technology research. Holds one active `ResearchProject` and up to 9 material slots. Players must deposit required materials before research begins. Right-clicking opens `ResearchBenchScreen`.

**Library** — A bookshelf-like block that can store up to 54 books (any `minecraft:writable_book` or `minecraft:written_book`). Passively emits a research speed aura in a 16-block radius. Players can read stored books to gain LITERACY XP.

**University** — A large block (3×3×4 multiblock structure) available from the Medieval age. Requires a `horizon_technologies:universities` tech unlock. Up to 4 players can be "enrolled" at once, gaining accelerated XP gain for Academic skills while within 8 blocks and holding any Horizon knowledge item.

**Laboratory** — Available from the Industrial age. Enables research into science-category technologies. Holds chemical reagent slots (12 slots) and an active experiment slot. Experiments have a success/failure probability modified by the player's SCIENCE skill level.

### 9.2 Apprenticeship System

Any NPC with a profession (registered in `HorizonRegistries.PROFESSIONS`) can serve as an apprenticeship master. Players can right-click an NPC master to enter an apprenticeship, which:

- Grants 1.5× skill XP for the NPC's associated skills while within 32 blocks
- Allows the NPC to teach recipes by right-clicking with relevant materials in hand
- Progresses through three stages: Apprentice (0–30 days), Journeyman (30–90 days), Master (90+ days)
- At Master stage, the player becomes a certified practitioner and can teach other players

### 11.1 WorldAgeData

```java
package com.horizonpack.horizoncore.data;

import net.minecraft.core.BlockPos;
import net.minecraft.nbt.*;
import net.minecraft.server.level.ServerLevel;
import net.minecraft.world.level.saveddata.SavedData;
import org.jetbrains.annotations.NotNull;

import java.util.*;

public class WorldAgeData extends SavedData {

    private static final String DATA_NAME = "horizon_world_age";

    // ── World state ───────────────────────────────────────────────────────────
    private int  currentDay  = 0;
    private long currentTick = 0;
    private HorizonAge globalAge = HorizonAge.STONE_AGE;

    // ── Settlement overrides ──────────────────────────────────────────────────
    // BlockPos (encoded as long) → days this settlement has accumulated
    private final Map<Long, Integer> settlementDays = new HashMap<>();

    // ── Civilization overrides ────────────────────────────────────────────────
    // Civilization UUID → days that civ has accumulated
    private final Map<UUID, Integer> civilizationDays = new HashMap<>();

    // ── Research bench progress (per bench position) ──────────────────────────
    // BlockPos long → active ResearchProject NBT
    private final Map<Long, ResearchProject> activeResearch = new HashMap<>();

    public WorldAgeData() {}

    // ── Static factory ────────────────────────────────────────────────────────
    public static WorldAgeData get(ServerLevel level) {
        return level.getDataStorage().computeIfAbsent(
            new SavedData.Factory<>(WorldAgeData::new, WorldAgeData::load),
            DATA_NAME
        );
    }

    // ── Load ──────────────────────────────────────────────────────────────────
    public static WorldAgeData load(CompoundTag tag) {
        WorldAgeData data = new WorldAgeData();
        data.currentDay  = tag.getInt("CurrentDay");
        data.currentTick = tag.getLong("CurrentTick");
        data.globalAge   = HorizonAge.values()[tag.getInt("GlobalAge")];

        ListTag settlements = tag.getList("Settlements", Tag.TAG_COMPOUND);
        for (Tag t : settlements) {
            CompoundTag s = (CompoundTag) t;
            data.settlementDays.put(s.getLong("Pos"), s.getInt("Days"));
        }

        ListTag civs = tag.getList("Civilizations", Tag.TAG_COMPOUND);
        for (Tag t : civs) {
            CompoundTag c = (CompoundTag) t;
            data.civilizationDays.put(c.getUUID("UUID"), c.getInt("Days"));
        }

        ListTag research = tag.getList("ActiveResearch", Tag.TAG_COMPOUND);
        for (Tag t : research) {
            CompoundTag r = (CompoundTag) t;
            long pos = r.getLong("Pos");
            ResearchProject project = new ResearchProject();
            project.deserializeNBT(r.getCompound("Project"));
            data.activeResearch.put(pos, project);
        }

        return data;
    }

    // ── Save ──────────────────────────────────────────────────────────────────
    @Override
    public @NotNull CompoundTag save(@NotNull CompoundTag tag) {
        tag.putInt("CurrentDay",  currentDay);
        tag.putLong("CurrentTick", currentTick);
        tag.putInt("GlobalAge",   globalAge.ordinal());

        ListTag settlements = new ListTag();
        settlementDays.forEach((pos, days) -> {
            CompoundTag s = new CompoundTag();
            s.putLong("Pos", pos); s.putInt("Days", days);
            settlements.add(s);
        });
        tag.put("Settlements", settlements);

        ListTag civs = new ListTag();
        civilizationDays.forEach((id, days) -> {
            CompoundTag c = new CompoundTag();
            c.putUUID("UUID", id); c.putInt("Days", days);
            civs.add(c);
        });
        tag.put("Civilizations", civs);

        ListTag research = new ListTag();
        activeResearch.forEach((pos, project) -> {
            CompoundTag r = new CompoundTag();
            r.putLong("Pos", pos);
            r.put("Project", project.serializeNBT());
            research.add(r);
        });
        tag.put("ActiveResearch", research);

        return tag;
    }

    // ── Day advancement ───────────────────────────────────────────────────────
    /**
     * Called by HorizonEventHandlers on LevelTickEvent every 24000 ticks.
     * Checks for age transitions and fires AgeAdvancementEvent if one occurs.
     */
    public void advanceDay(ServerLevel level) {
        currentDay++;
        currentTick += 24000;
        HorizonAge newAge = HorizonAge.fromDay(currentDay);
        if (newAge != globalAge) {
            HorizonAge oldAge = globalAge;
            globalAge = newAge;
            // Fire the event — all other mods listen to this
            net.neoforged.neoforge.common.NeoForge.EVENT_BUS.post(
                new com.horizonpack.horizoncore.events.custom.AgeAdvancementEvent(level, oldAge, newAge, null));
        }
        setDirty();
    }

    // ── Getters / setters ─────────────────────────────────────────────────────
    public int      getCurrentDay()         { return currentDay; }
    public long     getCurrentTick()        { return currentTick; }
    public HorizonAge getGlobalAge()        { return globalAge; }

    public HorizonAge getSettlementAge(BlockPos pos) {
        Integer days = settlementDays.get(pos.asLong());
        return days == null ? globalAge : HorizonAge.fromDay(days);
    }

    public void advanceSettlement(BlockPos pos) {
        settlementDays.merge(pos.asLong(), 1, Integer::sum);
        setDirty();
    }

    public HorizonAge getCivilizationAge(UUID civId) {
        return HorizonAge.fromDay(civilizationDays.getOrDefault(civId, 0));
    }

    public void advanceCivilization(UUID civId) {
        civilizationDays.merge(civId, 1, Integer::sum);
        setDirty();
    }

    public @Nullable ResearchProject getActiveResearch(BlockPos pos) {
        return activeResearch.get(pos.asLong());
    }

    public void setActiveResearch(BlockPos pos, ResearchProject project) {
        activeResearch.put(pos.asLong(), project);
        setDirty();
    }

    public void clearActiveResearch(BlockPos pos) {
        activeResearch.remove(pos.asLong());
        setDirty();
    }
}
```

### 12.2 Core Event Handlers

**File:** `events/HorizonEventHandlers.java`

```java
package com.horizonpack.horizoncore.events;

import com.horizonpack.horizoncore.capabilities.*;
import com.horizonpack.horizoncore.data.WorldAgeData;
import com.horizonpack.horizoncore.events.custom.DayAdvanceEvent;
import com.horizonpack.horizoncore.network.HorizonPacketHandler;
import com.horizonpack.horizoncore.network.packets.*;
import net.minecraft.server.level.ServerLevel;
import net.minecraft.server.level.ServerPlayer;
import net.minecraft.world.entity.player.Player;
import net.neoforged.bus.api.SubscribeEvent;
import net.neoforged.fml.common.Mod;
import net.neoforged.neoforge.common.NeoForge;
import net.neoforged.neoforge.event.entity.player.PlayerEvent;
import net.neoforged.neoforge.event.tick.LevelTickEvent;
import net.neoforged.neoforge.event.tick.PlayerTickEvent;

@Mod.EventBusSubscriber(modid = "horizoncore")
public class HorizonEventHandlers {

    /** Tick player capability every second (20 ticks). */
    @SubscribeEvent
    public static void onPlayerTick(PlayerTickEvent.Post event) {
        Player player = event.getEntity();
        if (player.level().isClientSide) return;
        if (player.tickCount % 20 != 0) return;

        IHorizonPlayerData data = HorizonCapabilities.get(player);
        if (data != null) {
            data.tick();
            // Sync HUD-relevant data to client every 5 seconds
            if (player.tickCount % 100 == 0 && player instanceof ServerPlayer sp) {
                HorizonPacketHandler.sendToPlayer(new SyncPlayerDataPacket(data.serializeNBT()), sp);
            }
        }
    }

    /** Copy capability data on player death + respawn. */
    @SubscribeEvent
    public static void onPlayerClone(PlayerEvent.Clone event) {
        if (!event.isWasDeath()) return;
        IHorizonPlayerData original = HorizonCapabilities.get(event.getOriginal());
        IHorizonPlayerData copy     = HorizonCapabilities.get(event.getEntity());
        if (original != null && copy != null) copy.copyFrom(original);
    }

    /** Sync capability to client on login. */
    @SubscribeEvent
    public static void onPlayerLogin(PlayerEvent.PlayerLoggedInEvent event) {
        if (!(event.getEntity() instanceof ServerPlayer sp)) return;
        IHorizonPlayerData data = HorizonCapabilities.get(sp);
        if (data != null) {
            HorizonPacketHandler.sendToPlayer(new SyncPlayerDataPacket(data.serializeNBT()), sp);
        }
    }

    /**
     * Level tick: advance world day counter every 24000 game ticks.
     * Also fires DayAdvanceEvent for sub-mods to run daily simulation.
     */
    @SubscribeEvent
    public static void onLevelTick(LevelTickEvent.Post event) {
        if (!(event.getLevel() instanceof ServerLevel level)) return;
        if (level.dimension() != ServerLevel.OVERWORLD) return;

        long tick = level.getGameTime();
        if (tick % 24000 == 0 && tick > 0) {
            WorldAgeData data = WorldAgeData.get(level);
            data.advanceDay(level);
            NeoForge.EVENT_BUS.post(new DayAdvanceEvent(level, data.getCurrentDay(), data.getGlobalAge()));
        }
    }

    /** Sync age to client when it changes. */
    @SubscribeEvent
    public static void onAgeAdvancement(
            com.horizonpack.horizoncore.events.custom.AgeAdvancementEvent event) {
        if (event.getLevel() instanceof ServerLevel sl) {
            sl.players().forEach(p ->
                HorizonPacketHandler.sendToPlayer(new SyncAgePacket(event.getNewAge()), p));
        }
    }
}
```

## 13. Network Packets

All packets use NeoForge's `RegisterPayloadHandlerEvent` system.

### 13.1 Packet Handler

**File:** `network/HorizonPacketHandler.java`

```java
package com.horizonpack.horizoncore.network;

import com.horizonpack.horizoncore.core.HorizonCore;
import com.horizonpack.horizoncore.network.packets.*;
import net.minecraft.server.level.ServerPlayer;
import net.neoforged.neoforge.network.PacketDistributor;
import net.neoforged.neoforge.network.event.RegisterPayloadHandlerEvent;
import net.neoforged.neoforge.network.registration.IPayloadRegistrar;

public class HorizonPacketHandler {

    public static void register(RegisterPayloadHandlerEvent event) {
        IPayloadRegistrar reg = event.registrar(HorizonCore.MODID).versioned("1");

        // Server → Client
        reg.play(SyncPlayerDataPacket.ID,    SyncPlayerDataPacket::new,
                 h -> h.client(SyncPlayerDataPacket::handleClient));
        reg.play(SyncAgePacket.ID,           SyncAgePacket::new,
                 h -> h.client(SyncAgePacket::handleClient));
        reg.play(TechUnlockPacket.ID,        TechUnlockPacket::new,
                 h -> h.client(TechUnlockPacket::handleClient));
        reg.play(SkillUpdatePacket.ID,       SkillUpdatePacket::new,
                 h -> h.client(SkillUpdatePacket::handleClient));

        // Client → Server
        reg.play(RequestResearchPacket.ID,   RequestResearchPacket::new,
                 h -> h.server(RequestResearchPacket::handleServer));
    }

    public static void sendToPlayer(Object packet, ServerPlayer player) {
        PacketDistributor.PLAYER.with(player).send(packet);
    }

    public static void sendToAllPlayers(Object packet, net.minecraft.server.MinecraftServer server) {
        PacketDistributor.ALL.noArg().send(packet);
    }

    public static void sendToServer(Object packet) {
        PacketDistributor.SERVER.noArg().send(packet);
    }
}
```

### 13.2 Packet Classes

| Packet | Direction | Purpose | Payload |
|--------|-----------|---------|---------|
| `SyncPlayerDataPacket` | S→C | Full capability sync | `CompoundTag` |
| `SyncAgePacket` | S→C | Notify client of age change | `int` (ordinal) |
| `TechUnlockPacket` | S→C | Trigger tech unlock animation + sound | `ResourceLocation` |
| `SkillUpdatePacket` | S→C | Update skill bar in HUD | `SkillType`, `int` level, `int` xp |
| `RequestResearchPacket` | C→S | Player clicks "Start Research" in GUI | `ResourceLocation` techId, `BlockPos` bench |

## 14. HUD & Client Rendering

### 14.1 HorizonHudOverlay

**File:** `client/gui/HorizonHudOverlay.java`

Registered via `RegisterGuiLayersEvent`. Draws:

- **Age badge** — top-right corner, shows current age name (e.g., "Iron Age")
- **Nutrition bars** — 6 bars (protein, carbs, fat, fiber, hydration, calories) drawn bottom-left above the hotbar. Each bar color-codes its status: green (healthy), yellow (low), red (critically low)
- **Body temperature** — thermometer icon next to nutrition bars; turns blue if hypothermic, red if hyperthermic
- **Active skill XP flash** — when a skill gains XP, a brief "+15 Metallurgy XP" toast appears

Client-side data is stored in a `ClientHorizonData` class populated by incoming packets.

### 14.2 TechTreeScreen

**File:** `client/gui/TechTreeScreen.java`

A full-screen scrollable/zoomable DAG viewer. Each tech node is rendered as a 48×48 icon button. Color coding:

- **Gray** — locked (prerequisites not met or age not reached)
- **Yellow** — available to research
- **Green** — already unlocked
- **Blue outline** — currently being researched

Clicking an available node opens a detail panel showing description, cost, time, and prerequisites. Clicking "Begin Research" sends `RequestResearchPacket`.

## 20. Critical Implementation Notes

### Thread Safety
- All `WorldAgeData` modifications happen server-side. Never call `WorldAgeData.get()` from a client thread.
- Always check `level.isClientSide()` before accessing server-only data.
- Enqueue work properly in packet handlers: `context.workHandler().submitAsync(() -> {...})`.
- `setDirty()` must be called on every `SavedData` mutation or changes will not persist.

### NBT Versioning
- Add a `"DataVersion"` int tag to every `serializeNBT()` output.
- In `deserializeNBT()`, check the version and migrate old formats before reading.
- Start at version 1. Increment whenever the NBT schema changes.

### Capability Persistence on Death
- `HorizonEventHandlers.onPlayerClone` copies all capability data on death.
- Skills, technologies, and language proficiency are always preserved.
- Nutrition levels are partially reset on death (configurable: `resetNutritionOnDeath`).

### Adding New Technologies from Sub-Mods
Sub-mods add technologies by creating their own `DeferredRegister<TechnologyData>` using the key `HorizonRegistries.TECHNOLOGY_REGISTRY_KEY`:

```java
// In your sub-mod's Registry class:
public static final DeferredRegister<TechnologyData> TECHNOLOGIES =
    DeferredRegister.create(HorizonRegistries.TECHNOLOGY_REGISTRY_KEY, MyMod.MODID);

public static final RegistryObject<TechnologyData> MY_TECH = TECHNOLOGIES.register("my_tech",
    () -> TechnologyData.builder(new ResourceLocation(MyMod.MODID, "my_tech"))
        .age(HorizonAge.IRON_AGE)
        .requires(new ResourceLocation("horizoncore", "iron_smelting"))
        .time(48000)
        .build());
```

### Adding New Skills from Sub-Mods
Similarly, sub-mods register `SkillDefinition` entries using `HorizonRegistries.SKILL_REGISTRY_KEY`. The `SkillType` enum must be extended if a truly new skill domain is needed — open an issue on the project repo to propose additions to the core enum.

### Performance
- The player capability tick runs every 20 ticks (1 second) per player. Keep `tick()` fast — no registry lookups, no level queries. Cache values.
- `TechnologyGraph.getAvailableTechs()` iterates all registered technologies. Cache the result per player per day; only invalidate when the player's tech set or the global age changes.
- `WorldAgeData` is loaded once and kept in memory. Do not call `computeIfAbsent` in hot paths; cache the reference in your event handler.
- Use `simulationRadius` from config to limit how many chunks are fully simulated at once. Chunks outside the radius run at 1/10th tick rate.

### AgeRecipeCondition Datageneration
When generating recipes programmatically (using `RecipeProvider`), emit the condition using:
```java
ConditionalRecipe.builder()
    .addCondition(new AgeRecipeCondition(HorizonAge.IRON_AGE))
    .addRecipe(/* your recipe */)
    .build(consumer, new ResourceLocation(MODID, "my_recipe"));
```
