## 1. Overview & Responsibilities

HorizonData is the **single material and crafting registry** for the entire Horizon modpack. It provides:

- **All ore blocks** — 25+ ore types with geologically accurate placement data  
- **All raw material items** — raw ores, ingots, dusts, crystals, fluids  
- **All block shapes** — slopes, arches, pillars, beams, carved, multi-material  
- **All wood and stone types** — 20+ wood variants, 15+ stone variants  
- **Composite and manufactured materials** — concrete, reinforced blocks, alloys  
- **The Fuel system** — per-fuel heat value, burn duration, age unlock, pollution output  
- **The Workshop system** — Forge, Carpentry Bench, Loom, Alchemy Lab, Kitchen, Tannery, Pottery Wheel, Bloomery, Crucible, Anvil, Grindstone, Workbench  
- **The Ore Processing Pipeline** — six-step refinement chain from raw ore → alloy  
- **The Structural Integrity system** — load-bearing calculations, collapse mechanics  
- **The Material Properties system** — melting point, hardness, density, conductivity, corrosion resistance, temperature resistance  
- **All crafting recipes** for workshops (custom `IRecipeType` per workshop)  
- **All tags** (`c:ores/`, `c:ingots/`, `c:dusts/`, `horizondata:` tags for age gating)  

Every other Horizon mod that needs materials, fuel, or crafting pulls from this mod's registries.

## 3. Mod Entry Point

**File:** `core/HorizonData.java`

```java
package com.horizonpack.horizondata.core;

import net.neoforged.bus.api.IEventBus;
import net.neoforged.fml.ModContainer;
import net.neoforged.fml.common.Mod;
import net.neoforged.fml.config.ModConfig;
import net.neoforged.fml.event.lifecycle.FMLCommonSetupEvent;
import net.neoforged.neoforge.common.NeoForge;

@Mod(HorizonData.MODID)
public class HorizonData {
    public static final String MODID = "horizondata";
    public static final String VERSION = "1.0.0";

    public HorizonData(IEventBus modEventBus, ModContainer modContainer) {
        modContainer.registerConfig(ModConfig.Type.COMMON, DataConfig.COMMON_SPEC);

        // Register all deferred registers
        DataRegistries.BLOCKS.register(modEventBus);
        DataRegistries.ITEMS.register(modEventBus);
        DataRegistries.BLOCK_ENTITIES.register(modEventBus);
        DataRegistries.MENU_TYPES.register(modEventBus);
        DataRegistries.RECIPE_TYPES.register(modEventBus);
        DataRegistries.RECIPE_SERIALIZERS.register(modEventBus);
        DataRegistries.CREATIVE_TABS.register(modEventBus);

        modEventBus.addListener(this::commonSetup);
        modEventBus.addListener(DataPacketHandler::register);

        NeoForge.EVENT_BUS.register(DataEventHandlers.class);
    }

    private void commonSetup(FMLCommonSetupEvent event) {
        event.enqueueWork(() -> {
            FuelRegistry.buildFuelMap();
            AlloyRegistry.buildAlloyMap();
            OreBlockRegistry.assignGeologicalData();
        });
    }
}
```

## 4. Ore & Material System

### 4.1 OreDefinition

**File:** `data/OreDefinition.java`

Each ore is described by a data object that drives both world-gen placement and the processing pipeline.

```java
package com.horizonpack.horizondata.data;

import com.horizonpack.horizoncore.data.HorizonAge;
import net.minecraft.resources.ResourceLocation;
import java.util.List;

public record OreDefinition(
    String oreId,                   // "iron", "copper", "tungsten" …
    OreCategory category,           // COMMON / UNCOMMON / RARE / VERY_RARE / FUEL
    GeologicalContext geology,       // where this ore forms geologically
    int veinSizeMin,                // minimum blocks per vein
    int veinSizeMax,                // maximum blocks per vein
    int veinsPerChunk,              // average veins per chunk (at peak depth)
    int depthMin,                   // minimum Y level
    int depthMax,                   // maximum Y level (inclusive)
    DepthDistribution distribution, // UNIFORM / TRIANGLE / TRAPEZOID / RARE_DEEP
    List<ResourceLocation> validBiomeTags,   // biome tags where this ore can generate
    List<ResourceLocation> invalidBiomeTags, // biome tags that exclude this ore
    HorizonAge ageUnlock,           // earliest age the player can meaningfully use this
    MaterialDefinition material,    // linked processed material data
    List<String> requiresGeology    // geological markers required ("volcanic", "kimberlite", …)
) {

    public enum OreCategory { COMMON, UNCOMMON, RARE, VERY_RARE, FUEL, GEM }

    public enum DepthDistribution {
        UNIFORM,       // equal chance from depthMin to depthMax
        TRIANGLE,      // peak in middle (like vanilla iron pre-1.18)
        TRAPEZOID,     // flat top, tapers at edges
        RARE_DEEP      // very low chance near surface, high chance at depthMin
    }

    public enum GeologicalContext {
        IGNEOUS,        // volcanic / intrusive rock regions
        SEDIMENTARY,    // ancient swamp / river delta deposits
        METAMORPHIC,    // high-pressure regions near mountains
        KIMBERLITE,     // ultra-deep volcanic pipe
        HYDROTHERMAL,   // near hot springs / ocean vents
        PLACER,         // river bed deposits
        EVAPORITE,      // dried lake beds (salt, potash)
        SUBSURFACE_OIL  // deep liquid pockets
    }
}
```

```java
package com.horizonpack.horizondata.data;

public record MaterialDefinition(
    String materialId,
    int meltingPointCelsius,    // determines which furnace tier can process it
    int hardness,               // 1-10 scale; determines required tool tier
    double densityKgPerM3,      // affects weight / transport cost
    double electricalConductivity,  // 0.0-1.0 scale; used for late-game wiring
    double thermalConductivity,     // 0.0-1.0 scale; affects temperature mechanics
    double corrosionResistance,     // 0.0-1.0 scale; affects weathering rate
    boolean isMagnetic,             // needed for motor/generator crafting
    int toxicity,               // 0-5 scale; affects handling safety
    double smeltYield,          // ingots per ore unit (0.5-3.0)
    int processingFurnaceTier   // 1=clay/stone, 2=brick, 3=blast, 4=electric arc
) {}
```

### 5.2 Ore Blocks

All 25 ores are registered with the pattern `<metal>_ore` (stone host) and `deepslate_<metal>_ore` (deepslate host below Y=0). Each uses `HorizonOreBlock` which stores the `OreDefinition` reference.

```java
// OreBlockRegistry.java — registration pattern
for (OreDefinition def : ALL_ORE_DEFINITIONS) {
    String id = def.oreId();
    DataRegistries.BLOCKS.register(id + "_ore",
        () -> new HorizonOreBlock(def, BlockBehaviour.Properties.of()
            .mapColor(MapColor.STONE)
            .requiresCorrectToolForDrops()
            .strength(3.0f, 3.0f)));

    DataRegistries.BLOCKS.register("deepslate_" + id + "_ore",
        () -> new HorizonOreBlock(def, BlockBehaviour.Properties.of()
            .mapColor(MapColor.DEEPSLATE)
            .requiresCorrectToolForDrops()
            .strength(4.5f, 3.0f)
            .sound(SoundType.DEEPSLATE)));
}
```

**Total ore blocks:** 50 (25 × stone host + 25 × deepslate host)  
**Special cases:**  
- `oil_deposit` — uses `OilDepositBlock` (fluid-like behaviour, requires pump to extract)

### 5.5 Slope Block Implementation

**File:** `blocks/shapes/SlopeBlock.java`

```java
package com.horizonpack.horizondata.blocks.shapes;

import net.minecraft.core.BlockPos;
import net.minecraft.core.Direction;
import net.minecraft.world.item.context.BlockPlaceContext;
import net.minecraft.world.level.BlockGetter;
import net.minecraft.world.level.block.Block;
import net.minecraft.world.level.block.state.BlockState;
import net.minecraft.world.level.block.state.StateDefinition;
import net.minecraft.world.level.block.state.properties.*;
import net.minecraft.world.phys.shapes.CollisionContext;
import net.minecraft.world.phys.shapes.VoxelShape;
import net.minecraft.world.phys.shapes.Shapes;

public class SlopeBlock extends Block {

    // Angles: 15, 30, 45, 60, 75 degrees
    public static final EnumProperty<SlopeAngle> ANGLE =
        EnumProperty.create("angle", SlopeAngle.class);
    public static final DirectionProperty FACING =
        HorizontalDirectionalBlock.FACING;
    public static final BooleanProperty UPSIDEDOWN =
        BlockStateProperties.HALF; // TOP = upside-down, BOTTOM = normal

    // VoxelShape lookup table [ANGLE_INDEX][FACING_INDEX]
    private static final VoxelShape[][] SHAPES = buildShapes();

    public SlopeBlock(Properties properties) {
        super(properties);
        this.registerDefaultState(this.stateDefinition.any()
            .setValue(ANGLE, SlopeAngle.DEG_45)
            .setValue(FACING, Direction.NORTH)
            .setValue(UPSIDEDOWN, false));
    }

    @Override
    protected void createBlockStateDefinition(StateDefinition.Builder<Block, BlockState> builder) {
        builder.add(ANGLE, FACING, UPSIDEDOWN);
    }

    @Override
    public VoxelShape getShape(BlockState state, BlockGetter level, BlockPos pos, CollisionContext context) {
        int angleIdx = state.getValue(ANGLE).ordinal();
        int facingIdx = state.getValue(FACING).get2DDataValue();
        boolean flipped = state.getValue(UPSIDEDOWN);
        VoxelShape base = SHAPES[angleIdx][facingIdx];
        return flipped ? flipVertical(base) : base;
    }

    @Override
    public BlockState getStateForPlacement(BlockPlaceContext context) {
        Direction facing = context.getHorizontalDirection().getOpposite();
        boolean top = context.getClickedFace() == Direction.DOWN ||
            (context.getClickedFace() != Direction.UP &&
             context.getClickLocation().y - context.getClickedPos().getY() > 0.5);
        return this.defaultBlockState()
            .setValue(FACING, facing)
            .setValue(UPSIDEDOWN, top);
    }

    public enum SlopeAngle implements StringRepresentable {
        DEG_15("15"), DEG_30("30"), DEG_45("45"), DEG_60("60"), DEG_75("75");
        private final String name;
        SlopeAngle(String name) { this.name = name; }
        @Override public String getSerializedName() { return name; }
    }

    private static VoxelShape[][] buildShapes() {
        // 5 angles × 4 horizontal directions
        VoxelShape[][] shapes = new VoxelShape[5][4];
        // DEG_45, NORTH example:
        shapes[2][0] = Shapes.or(
            Block.box(0,0,0, 16,8,16),      // bottom half solid
            Block.box(0,8,0, 16,16,8)       // back-half top
        );
        // ... (all 20 combinations computed similarly)
        return shapes;
    }

    private static VoxelShape flipVertical(VoxelShape s) {
        // Mirror shape around Y=8 axis
        return s.move(0, -1, 0); // simplified; real impl uses Shapes.join
    }
}
```

```java
    String fuelId,
    FuelType type,

    public enum PollutionLevel {
        NONE(0), LOW(10), MEDIUM(25), HIGH(50), VERY_HIGH(75), RADIATION(100);
        public final int value;
        PollutionLevel(int v) { this.value = v; }
    }

    // Max temperatures per fuel type:
    // dried_dung      → 400°C   (pottery, lime burning only)
    // wood_log        → 600°C   (copper smelting)
    // coal_chunk      → 900°C   (iron smelting fails; needs coke or blast)
```

```java
package com.horizonpack.horizondata.recipes;

import com.mojang.serialization.MapCodec;
import com.mojang.serialization.codecs.RecordCodecBuilder;
import net.minecraft.core.HolderLookup;
import net.minecraft.network.RegistryFriendlyByteBuf;
import net.minecraft.network.codec.StreamCodec;
import net.minecraft.world.item.ItemStack;
import net.minecraft.world.item.crafting.*;
import net.minecraft.world.level.Level;

public class ForgeRecipe implements Recipe<ForgeRecipeInput> {

    private final Ingredient fuel;          // acceptable fuel ingredient
    private final Ingredient input;         // ore / metal item to process
    private final ItemStack result;         // output stack
    private final int minHeatCelsius;       // minimum forge temperature needed
    private final int processingTicks;      // how many ticks the recipe runs
    private final Ingredient toolRequired;  // e.g. tongs, hammer — null if none
    private final boolean requiresAnvil;    // true for shaping recipes (plate, rod)
    private final String requiredAge;       // "iron_age" etc.; checked via HorizonCore cap

    public static final MapCodec<ForgeRecipe> CODEC = RecordCodecBuilder.mapCodec(inst -> inst.group(
        Ingredient.CODEC.fieldOf("fuel").forGetter(r -> r.fuel),
        Ingredient.CODEC.fieldOf("input").forGetter(r -> r.input),
        ItemStack.CODEC.fieldOf("result").forGetter(r -> r.result),
        net.minecraft.util.ExtraCodecs.NON_NEGATIVE_INT.fieldOf("min_heat_celsius").forGetter(r -> r.minHeatCelsius),
        net.minecraft.util.ExtraCodecs.NON_NEGATIVE_INT.fieldOf("processing_ticks").forGetter(r -> r.processingTicks),
        Ingredient.CODEC.optionalFieldOf("tool_required", Ingredient.EMPTY).forGetter(r -> r.toolRequired),
        net.minecraft.util.ExtraCodecs.strictOptionalField(net.minecraft.util.ExtraCodecs.LITERAL_STRING_CODEC, "required_age", "").forGetter(r -> r.requiredAge)
    ).apply(inst, (fuel, input, result, heat, ticks, tool, age) ->
        new ForgeRecipe(fuel, input, result, heat, ticks, tool, false, age)));

    public static final StreamCodec<RegistryFriendlyByteBuf, ForgeRecipe> STREAM_CODEC = /* ... */;

    @Override
    public boolean matches(ForgeRecipeInput input, Level level) {
        return this.input.test(input.getInputItem())
            && this.fuel.test(input.getFuelItem())
            && (this.toolRequired == Ingredient.EMPTY || this.toolRequired.test(input.getToolItem()))
            && input.getCurrentHeat() >= this.minHeatCelsius;
    }

    @Override
    public ItemStack assemble(ForgeRecipeInput input, HolderLookup.Provider registries) {
        return result.copy();
    }

    @Override
    public RecipeSerializer<ForgeRecipe> getSerializer() {
        return DataRegistries.FORGE_RECIPE_SERIALIZER.get();
    }

    @Override
```

### 8.4 Multi-Step Sword Example

To craft an iron sword via the workshop system requires these sequential recipes:

```
1. [grindstone_recipe] iron_ore → iron_dust (x2)
2. [bloomery_recipe]   iron_dust + charcoal → pig_iron_ingot (requires 1200°C)
3. [forge_recipe]      pig_iron_ingot + coke_coal → iron_ingot (requires 1400°C)
4. [forge_recipe]      iron_ingot → iron_billet (hammer + tongs required)
5. [anvil_recipe]      iron_billet → iron_blade (hammer required, 3 hits)
6. [grindstone_recipe] iron_blade → sharpened_iron_blade
7. [workbench_recipe]  sharpened_iron_blade + stick → iron_sword
```

This multi-step process is tracked by the player's `IHorizonPlayerData.getCraftingChainProgress()` capability field from HorizonCore.

### 10.1 ForgeBlockEntity

**File:** `blockentities/ForgeBlockEntity.java`

```java
package com.horizonpack.horizondata.blockentities;

import net.minecraft.core.BlockPos;
import net.minecraft.nbt.CompoundTag;
import net.minecraft.world.SimpleContainer;
import net.minecraft.world.item.ItemStack;
import net.minecraft.world.item.crafting.RecipeHolder;
import net.minecraft.world.level.block.entity.BlockEntity;
import net.minecraft.world.level.block.state.BlockState;

public class ForgeBlockEntity extends BlockEntity implements MenuProvider {

    // Slots: 0=fuel, 1=input, 2=tool, 3=output
    private final SimpleContainer inventory = new SimpleContainer(4);

    private int currentHeat = 0;        // current temperature in °C
    private int targetHeat = 0;         // temperature this fuel will reach
    private int fuelBurnTicks = 0;      // ticks remaining on current fuel
    private int processProgress = 0;    // ticks into current recipe
    private int processTotal = 0;       // total ticks recipe requires
    private RecipeHolder<ForgeRecipe> currentRecipe = null;

    public static final int HEAT_RISE_RATE = 5;    // °C per tick while burning
    public static final int HEAT_DROP_RATE = 2;    // °C per tick when not burning
    public static final int MAX_HEAT = 3500;        // electric arc maximum

    public ForgeBlockEntity(BlockPos pos, BlockState state) {
        super(DataRegistries.FORGE_BLOCK_ENTITY.get(), pos, state);
    }

    public static void serverTick(Level level, BlockPos pos, BlockState state, ForgeBlockEntity be) {
        boolean wasBurning = be.fuelBurnTicks > 0;

        // 1. Manage fuel consumption
        if (be.fuelBurnTicks <= 0 && be.currentRecipe != null) {
            ItemStack fuelStack = be.inventory.getItem(0);
            FuelDefinition fuel = FuelRegistry.getFuel(fuelStack);
            if (fuel != null && fuelStack.getCount() > 0) {
                be.fuelBurnTicks = fuel.burnDurationTicks();
                be.targetHeat = fuel.maxTemperatureCelsius();
                fuelStack.shrink(1);
            }
        }

        // 2. Update heat
        if (be.fuelBurnTicks > 0) {
            be.fuelBurnTicks--;
            if (be.currentHeat < be.targetHeat) {
                be.currentHeat = Math.min(be.currentHeat + HEAT_RISE_RATE, be.targetHeat);
            }
        } else {
            be.targetHeat = 0;
            if (be.currentHeat > 0) {
                be.currentHeat = Math.max(0, be.currentHeat - HEAT_DROP_RATE);
            }
        }

        // 3. Update blockstate LIT and HEAT_LEVEL
        boolean isLit = be.fuelBurnTicks > 0;
        int heatLevel = be.currentHeat >= 1400 ? 4 : be.currentHeat >= 900 ? 3 :
                        be.currentHeat >= 600 ? 2 : be.currentHeat >= 200 ? 1 : 0;
        BlockState newState = state.setValue(ForgeBlock.LIT, isLit)
                                   .setValue(ForgeBlock.HEAT_LEVEL, heatLevel);
        if (!newState.equals(state)) level.setBlock(pos, newState, 3);

        // 4. Process recipe
        if (be.currentRecipe != null) {
            ForgeRecipeInput input = new ForgeRecipeInput(
                be.inventory.getItem(1), be.inventory.getItem(0),
                be.inventory.getItem(2), be.currentHeat);

            if (be.currentRecipe.value().matches(input, level)) {
                be.processProgress++;
                if (be.processProgress >= be.processTotal) {
                    // Output result
                    ItemStack result = be.currentRecipe.value().assemble(input, level.registryAccess());
                    be.inventory.setItem(3, result);
                    be.inventory.getItem(1).shrink(1);
                    be.processProgress = 0;
                    // Fire event
                    NeoForge.EVENT_BUS.post(new WorkshopCraftEvent(be, be.currentRecipe, result));
                }
            } else {
                be.processProgress = 0; // heat dropped below threshold
            }
        }

        // 5. Sync to clients
        if (isLit != wasBurning) {
            DataPacketHandler.sendToNearby(level, pos, new SyncWorkshopPacket(pos, be));
        }
    }

    @Override
    public void saveAdditional(CompoundTag tag, HolderLookup.Provider registries) {
        super.saveAdditional(tag, registries);
        tag.putInt("heat", currentHeat);
        tag.putInt("fuel_ticks", fuelBurnTicks);
        tag.putInt("progress", processProgress);
        tag.putInt("progress_total", processTotal);
        ContainerHelper.saveAllItems(tag, inventory.getItems(), registries);
    }

    @Override
    public void loadAdditional(CompoundTag tag, HolderLookup.Provider registries) {
        super.loadAdditional(tag, registries);
        currentHeat    = tag.getInt("heat");
        fuelBurnTicks  = tag.getInt("fuel_ticks");
        processProgress = tag.getInt("progress");
        processTotal   = tag.getInt("progress_total");
        ContainerHelper.loadAllItems(tag, inventory.getItems(), registries);
    }

    // MenuProvider
    @Override
    public AbstractContainerMenu createMenu(int containerId, Inventory playerInv, Player player) {
        return new ForgeMenu(containerId, playerInv, this);
    }
    @Override
    public Component getDisplayName() { return Component.translatable("block.horizondata.forge"); }
}
```

## 11. Structural Integrity System

### 11.1 Overview

Every `HorizonBlock` derivative (stone, wood, composite, metal) carries a `structuralLoad` value. When a block is placed, `StructuralIntegrity.checkSupport()` runs a flood-fill from that block downward, verifying that the accumulated load is within the support capacity of load-bearing columns and floor blocks.

### 11.2 Load Values

| Block Category | Load Units |
|---|---|
| Air | 0 |
| Wood planks / slab | 1 |
| Wood beam / log | 2 |
| Stone (light: sandstone, pumice) | 4 |
| Stone (standard: limestone, granite) | 6 |
| Stone (heavy: basalt, quartzite) | 8 |
| Metal block (iron, steel) | 10 |
| Composite (concrete) | 7 |
| Reinforced concrete | 12 |
| Filled chest / barrel | +3 |
| Entity standing on block | +1 per entity |

### 11.3 Support Capacity

| Support Element | Max Load Units | Notes |
|---|---|---|
| Wood pillar (oak) | 30 | Buckles if load > 30 |
| Stone pillar (limestone) | 80 | |
| Stone pillar (granite) | 150 | |
| Iron beam | 200 | |
| Steel I-beam | 500 | |
| Arch (distributes load) | ×2 capacity of material | Flying buttress effect |
| Foundation block (stone) | Unlimited | Must connect to bedrock chain |

### 11.4 Collapse Mechanic

```java
// StructuralIntegrity.java
public class StructuralIntegrity {

    /** Called on BlockPlaceEvent and via periodic server tick (every 200 ticks). */
    public static void checkSupport(Level level, BlockPos placedPos) {
        int totalLoad = calculateColumnLoad(level, placedPos);
        int capacity = getSupportCapacity(level, placedPos);

        if (totalLoad > capacity) {
            // Trigger collapse event
            StructureCollapseEvent event = new StructureCollapseEvent(level, placedPos, totalLoad, capacity);
            NeoForge.EVENT_BUS.post(event);
            if (!event.isCanceled()) {
                performCollapse(level, placedPos);
            }
        }
    }

    private static void performCollapse(Level level, BlockPos origin) {
        // BFS from origin upward — convert load-bearing blocks to falling block entities
        Queue<BlockPos> queue = new ArrayDeque<>();
        queue.add(origin);
        Set<BlockPos> visited = new HashSet<>();
        while (!queue.isEmpty()) {
            BlockPos pos = queue.poll();
            if (visited.contains(pos) || !isLoadBearing(level.getBlockState(pos))) continue;
            visited.add(pos);
            // Convert to falling block
            FallingBlockEntity fb = FallingBlockEntity.fall(level, pos, level.getBlockState(pos));
            fb.dropItem = true; // drops as item when lands
            level.setBlock(pos, Blocks.AIR.defaultBlockState(), 3);
            // Propagate to blocks above
            queue.add(pos.above());
        }
        level.playSound(null, origin, SoundEvents.STRUCTURAL_INTEGRITY_COLLAPSE,
            SoundSource.BLOCKS, 1.5f, 0.8f);
    }
}
```

## 12. Material Properties System

### 12.1 WeatheringSystem

**File:** `systems/WeatheringSystem.java`

Blocks that can weather (iron, copper, bronze, certain stone types) track a `weatherStage` from 0 to 4 stored in the block's NBT. Every `WEATHERING_TICK_INTERVAL` (default 24000 ticks = 1 in-game day) the system checks exposure to rain, humidity, and temperature.

```java
public class WeatheringSystem {

    public static final int WEATHERING_TICK_INTERVAL = 24000;
    public static final int MAX_WEATHER_STAGE = 4;

    // Weather stage descriptions per material:
    // IRON:   0=clean, 1=tarnish, 2=light_rust, 3=heavy_rust, 4=corroded
    // COPPER: 0=clean, 1=tarnish, 2=patina_start, 3=patina, 4=verdigris
    // STONE:  0=clean, 1=mossy_start, 2=mossy, 3=weathered, 4=eroded
    // BRONZE: 0=clean, 1=tarnish, 2=oxidized, 3=heavily_oxidized, 4=patina

    public static void tickWeathering(ServerLevel level, BlockPos pos, BlockState state) {
        if (!(state.getBlock() instanceof WeatheringBlock wb)) return;
        if (!shouldWeather(level, pos)) return;

        CompoundTag nbt = getBlockNbt(level, pos);
        int stage = nbt.getInt("weather_stage");
        if (stage >= MAX_WEATHER_STAGE) return;

        float weatherChance = calculateWeatherChance(level, pos);
        if (level.random.nextFloat() < weatherChance) {
            nbt.putInt("weather_stage", stage + 1);
            setBlockNbt(level, pos, nbt);
            // Trigger texture update via model data update
            level.sendBlockUpdated(pos, state, state, 3);
            NeoForge.EVENT_BUS.post(new MaterialWeatheredEvent(level, pos, stage, stage + 1));
        }
    }

    private static float calculateWeatherChance(ServerLevel level, BlockPos pos) {
        float base = 0.01f;                              // 1% per day base chance
        if (level.isRaining() && level.canSeeSky(pos)) base *= 3f;    // 3% in rain
        if (level.getBiome(pos).value().getBaseTemperature() > 0.7f) base *= 1.5f; // humid
        if (level.getBiome(pos).value().getBaseTemperature() < 0.0f) base *= 0.5f; // frozen
        return base;
    }
}
```

### 12.2 Visual Weathering — Model Data Override

Weathering stages are rendered via `IClientItemExtensions.getModelData()` which injects a `ModelProperty<Integer>` (`WEATHER_STAGE_PROPERTY`) into the block's BakedModel lookup, selecting from 5 pre-baked model variants per material.

```java
@EventBusSubscriber(modid = HorizonData.MODID, bus = EventBusSubscriber.Bus.GAME)
public class DataEventHandlers {

    @SubscribeEvent
    public static void onFuelBurn(FurnaceFuelBurnTimeEvent event) {
        int burnTime = FuelRegistry.getVanillaBurnTime(event.getItemStack());
        if (burnTime > 0) event.setBurnTime(burnTime);
    }

    @SubscribeEvent
    public static void onBlockBreak(BlockBreakEvent event) {
        Block block = event.getState().getBlock();
        if (block instanceof HorizonOreBlock oreBlock) {
            OreDefinition def = oreBlock.getOreDef();
            OreMinedEvent oreEvent = new OreMinedEvent(
                event.getPlayer(), event.getPos(), def, ProcessingStage.RAW);
            NeoForge.EVENT_BUS.post(oreEvent);
            if (oreEvent.isCanceled()) event.setCanceled(true);
        }
    }

    @SubscribeEvent
    public static void onServerTick(LevelTickEvent.Post event) {
        if (!(event.getLevel() instanceof ServerLevel level)) return;
        if (level.getGameTime() % WeatheringSystem.WEATHERING_TICK_INTERVAL == 0) {
            // Iterate all loaded chunks and tick weathering on exposed blocks
            WeatheringSystem.tickAllLoadedChunks(level);
        }
        if (level.getGameTime() % StructuralIntegrity.CHECK_INTERVAL == 0) {
            StructuralIntegrity.periodicCheck(level);
        }
    }

    @SubscribeEvent
    public static void onBlockPlace(BlockEvent.EntityPlaceEvent event) {
        if (DataConfig.COMMON.enableStructuralIntegrity.get()) {
            StructuralIntegrity.checkSupport(event.getLevel(), event.getPos());
        }
    }
}
```

## 24. Critical Implementation Notes

### 24.1 Non-Cubic Block Rendering
Slope, arch, pillar, and beam blocks all use `VoxelShape`-based collision and custom `BakedModel` implementations. They must be registered as `RenderType.cutout()` or `translucent` where applicable — set via `ItemBlockRenderTypes.setRenderLayer(block, RenderType.cutout())` in `@OnlyIn(Dist.CLIENT)` setup.

### 24.2 Heat-Gated Recipes & Age Locking
`ForgeRecipe.matches()` calls back into HorizonCore's `IHorizonPlayerData` capability via `HorizonCore.getPlayerData(player)` to verify the player's current age meets `required_age`. If HorizonCore is absent (dependency mismatch) this check is bypassed with a warning log — **never crash on missing soft dependency.**

### 24.3 Ore Processing and Fortune Enchant
`OreMinedEvent` is fired before the vanilla loot table drop. If `OreMinedEvent` is not canceled, vanilla drops proceed. To avoid double-dropping, `HorizonOreBlock` overrides `getDrops()` to return an empty list when `OreMinedEvent` is handled, and the event handler injects the correct `ProcessingStage.RAW` item with Fortune applied via `EnchantmentHelper.getItemEnchantmentLevel`.

### 24.4 Structural Integrity Performance
Running `StructuralIntegrity.checkSupport()` on every single block place would be too expensive. The check is:
- **Immediate** only for blocks in `#horizondata:load_bearing` tag placed at Y > 60 (surface structures)
- **Deferred** for underground placement — added to a priority queue checked on `collapseCheckInterval` ticks
- **Skipped entirely** if `enableStructuralIntegrity = false` in config

### 24.5 Weathering and Saved Data
Block weathering state is stored in `BlockEntityTag` NBT, not block state properties. This means weathering blocks must be `BaseEntityBlock` subclasses with a minimal `BlockEntity` that only stores the `weather_stage` integer. The `WeatheringBlockEntity` is extremely lightweight (8 bytes NBT) and uses `LazyOptional` to avoid memory pressure in large structures.

### 24.6 Alloy Ratios in Crucible
The `CrucibleBlockEntity` stores four float slots `mixRatios[0..3]` tracking how many ingot-equivalents of each metal are currently molten. When the total ratio matches a registered `AlloyFormula` within ±5% tolerance (configurable), the output triggers. This tolerancing prevents players from needing to be pixel-perfect in their ratio measurement.

### 24.7 Oil and Gas Deposit Blocks
`OilDepositBlock` does not behave like a standard block. It:
- Uses a custom `BlockEntity` (`OilDepositBlockEntity`) tracking reservoir volume (in mB)
- Cannot be mined directly — requires `oil_pump` block (registered in HorizonPower)
- Emits `OreMinedEvent` with `ProcessingStage.RAW` and item type `oil_bucket` when pump extracts
- Has `GasDepositBlock` sibling that plays explosion sound + spawns fire particles if ignited while uncontained

### 24.8 Creative Tabs
HorizonData ships 4 creative tabs:
- **Ores & Materials** — all ore blocks, raw ores, ingots, dusts, nuggets, plates, rods, wires, gears
- **Blocks & Shapes** — all wood types, stone types, slopes, arches, pillars, beams, composites
- **Workshop Stations** — all 12 workshop blocks + their tool items
- **Fuels & Energy** — all fuel items, fuel storage blocks

### 24.9 Data Generation
`DataGenerator` classes exist for:
- `OreBlockStateProvider` — generates all 50 ore blockstate + model JSONs automatically
- `WoodBlockStateProvider` — generates all 21 × 20 wood variant blockstate + model JSONs
- `StoneBlockStateProvider` — generates all 15 × 13 stone variant blockstate + model JSONs
- `OreRecipeProvider` — generates all grindstone + forge + bloomery + crucible recipe JSONs for all metals
- `OreLootTableProvider` — generates all 50 ore loot tables
- `TagProvider` — generates all c: namespace tags

Run via `./gradlew runData` before building.
